# Taskpad Web Deploy

A static PWA. Deploy to any host that serves files over HTTPS (Cloudflare Pages, Netlify, Vercel, etc.).

## Files to deploy
```
index.html
app.js
app/sync-core.mjs
manifest.json
sw.js
config.json
icon-192.png    <- replace with your own 192×192 icon
icon-512.png    <- replace with your own 512×512 icon
fonts.css       <- optional, generated by `node download-fonts.mjs`
fonts/          <- optional, generated by `node download-fonts.mjs`
```

## config.json
Set your Cloudflare Worker URL before deploying. There is no shared default endpoint.
```json
{ "workerUrl": "https://your-worker.workers.dev" }
```
If `workerUrl` is empty or absent the app runs in local-only mode (no sync).

## Optional: offline fonts
Run from the tray project root:
```
node download-fonts.mjs
```
Then copy `src/fonts.css` and `src/fonts/` into this web folder.
Without it the app loads fonts from Google Fonts and requires internet for first load.

## Service worker cache
The service worker caches `index.html`, `app.js`, `app/sync-core.mjs`, and `manifest.json`.
It never caches `config.json` so Worker URL changes take effect immediately on next load.

**After every deployment:** bump `CACHE_VERSION` in `sw.js` (e.g. `taskpad-2025-07-01`).
If you forget, returning users will see stale cached assets until their browser evicts the old cache.
